What is Intervals?

Intervals is a robust application designed to schedule and trigger notifications at specified times in the future with exceptional performance and reliability.

High Performance

Minimal latency delivery with optimized processing for time-critical notifications.

High Throughput

Handles large volumes of concurrent notifications with ease.

Scalable

Horizontal scaling from small workloads to enterprise-level deployments.

Reliable

Built on Kafka + MongoDB with distributed locking for fault tolerance.

Flexible Scheduling

ISO 8601 support - single, duration, and periodic scheduling options.

Time-Sensitive

Perfect for reminders, alerts, promotional campaigns, and workflow automation.

How It Works

Distributed microservices architecture with Kafka and MongoDB

└─────────────────┘     └──────────────────┘     └────────┬────────┘
                                                         │
                                                ┌────────▼────────┐
                                                │   MongoDB       │

                                                         │
┌─────────────────┐     ┌──────────────────┐     ┌────────▼────────┐
│   Your App      │◀────│  Kafka Topic     │◀────│ Notification    │

Microservices

Service Purpose Port
assignment-provider Consumes assignments, persists to MongoDB 8080
execution-producer Creates execution records from schedules 8081
notification-executor Executes notifications at scheduled time 8082
topic-aggregator Aggregates Kafka topics (Streams) -
topic-distributor Distributes topics (Streams) -
record-archiver Archives completed records -

Data Flow

1. Send

Publish to assignments.t30600 with schedule header

2. Store

Assignment Provider persists to MongoDB

3. Schedule

Execution Producer creates execution records

4. Execute

Notification Executor fires at the right time

5. Deliver

Message appears on executions.t30600

Quick Integration

Get started in minutes with just a few lines of code

Step 1: Send a Scheduled Message

val producer = KafkaProducer<String, ByteArray>(properties)

val record = ProducerRecord<String, ByteArray>(
    "assignments.t30600",
    payload
)

record.headers().add("intervals-schedule", "2023-07-16T19:20:30.45Z")
record.headers().add("intervals-tenant-id", "your-tenant-id")

producer.send(record)

Step 2: Receive the Execution

consumer.subscribe(listOf("executions.t30600"))

// When the scheduled time arrives,
// the message will be delivered here
// with all original headers preserved

Headers Reference

Header Required Description
intervals-schedule Yes ISO 8601 schedule
intervals-assignment-id No UUID (auto-generated if omitted)
intervals-tenant-id No For multi-tenancy

Schedule Formats

Intervals uses ISO 8601 based scheduling for maximum flexibility

Single Execution

Execute once at the specified time.

2023-07-16T19:20:30.45Z

Duration (from now)

Execute at a specified duration from now.

PT7H

Periodic (recurring)

Execute every interval from start to end date.

R/2023-07-16T19:20:30.45Z/PT7H/2023-07-17T20:20:30.45Z
